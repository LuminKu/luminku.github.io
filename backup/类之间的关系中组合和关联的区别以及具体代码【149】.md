在面向对象编程中，组合（Composition） 和 关联（Association） 是两种描述类之间关系的重要机制，它们的核心区别在于 生命周期依赖性 和 语义强度。

---

核心区别

- 关联（Association）  
  - 表示两个类之间存在“使用”或“引用”关系，但彼此独立存在。  
  - 生命周期互不影响：一个对象销毁，另一个仍可存在。  
  - 语义较弱，属于“has-a”但不强占有的关系。

- 组合（Composition）  
  - 表示“整体-部分”的强依赖关系，部分不能脱离整体而存在。  
  - 生命周期绑定：整体销毁时，部分也必须被销毁。  
  - 语义更强，属于“contains-a”或“owns-a”的关系。

> ✅ 关键判断标准：如果部分对象离开整体就失去意义或无法存在 → 是组合；否则是关联（或聚合）。

---

具体代码示例（Java）

1. 关联关系示例
```java
class Teacher {
    private String name;
    private List<Student> students; // 持有Student引用，但Student可独立存在

    public Teacher(String name) {
        this.name = name;
        this.students = new ArrayList<>();
    }

    public void addStudent(Student student) {
        students.add(student);
    }
}

class Student {
    private String name;

    public Student(String name) {
        this.name = name;
    }
}

// 使用示例
Student s = new Student("Alice");
Teacher t = new Teacher("Mr. Smith");
t.addStudent(s);

// 即使删除Teacher，Student仍可存在
t = null;
System.out.println(s.name); // 输出: Alice → 依然有效
```

> ✅ 特点：`Teacher` 持有 `Student` 引用，但 `Student` 对象不依赖 `Teacher` 的生命周期。

---

2. 组合关系示例
```java
class Room {
    private String name;

    public Room(String name) {
        this.name = name;
    }

    // Room 的生命周期由 House 控制，不能独立存在
}

class House {
    private List<Room> rooms; // House 组合了 Room

    public House() {
        rooms = new ArrayList<>();
        rooms.add(new Room("客厅")); // House 在构造时创建 Room
        rooms.add(new Room("卧室"));
    }

    // 当 House 被销毁，所有 Room 也应被销毁（Java 中由 GC 自动处理）
}
```

> ✅ 特点：`Room` 对象由 `House` 创建并管理，语义上“房间不能脱离房子而存在”。

---

补充说明：关联 vs 聚合 vs 组合

| 关系类型 | 生命周期 | 代码表现 | 典型场景 |
|----------|----------|----------|----------|
| 关联 | 独立 | 成员变量引用，对象可外部传入 | 教师-学生、客户-订单 |
| 聚合 | 独立 | 成员变量，通常通过 Setter 注入 | 部门-员工、汽车-轮胎 |
| 组合 | 绑定 | 成员变量在构造函数中创建 | 房子-房间、公司-部门（若部门随公司注销） |

> 💡 注意：在 Java 等语言中，关联、聚合、组合的代码写法可能相同（都是成员变量），区别在于语义和设计意图。

---

总结
- 若对象可独立存在 → 用 关联 或 聚合。  
- 若对象必须依附于整体 → 用 组合。  
- 组合强调 “同生共死”，关联强调 “彼此认识但各自独立”。
