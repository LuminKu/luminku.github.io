核心结论

同一局域网内可互相通信的IP是10.202.24.209（无线局域网适配器WLAN），这是当前主机实际接入物理局域网的有效IP，其余IP均为虚拟网卡/本地回环类IP，仅用于本地虚拟化/专属场景，无法和局域网内其他物理电脑通信。

各IP段详细说明

网卡适配器 IPv4地址 子网掩码 用途&通信范围
VMware Network Adapter VMnet1 192.168.110.1 255.255.255.0 VMware仅主机模式虚拟网卡，仅用于主机和VMnet1网段的虚拟机通信，与物理局域网隔离
VMware Network Adapter VMnet8 192.168.33.1 255.255.255.0 VMwareNAT模式虚拟网卡，仅用于主机和VMnet8网段的虚拟机通信，虚拟机通过此网卡做网络地址转换上网，与物理局域网隔离
WLAN（无线） 10.202.24.209 255.254.0.0 物理局域网有效IP，主机通过无线接入路由器/交换机组成的局域网，同一网段（10.202.0.0~10.203.255.255）的其他电脑可通过此IP通信
本地连接*4 192.168.137.1 255.255.255.0 Windows移动热点/ICS（Internet连接共享） 虚拟网卡，仅用于主机共享网络给其他设备（如手机、平板），与物理局域网隔离

补充验证方法

若要确认局域网内其他电脑是否能和此主机通信，可执行2步操作：

  1. 主机端打开命令提示符，输入 ping 目标电脑IP（目标电脑需和主机连同一WiFi/网线），能收到回复则通信正常；
  2. 目标电脑端输入 ping 10.202.24.209，能收到回复则双向通信正常。

注意点

如果ping不通，大概率是Windows防火墙拦截了ICMP请求，可临时关闭防火墙测试，或在防火墙中放行「文件和打印机共享」「ICMPv4入站请求」。
