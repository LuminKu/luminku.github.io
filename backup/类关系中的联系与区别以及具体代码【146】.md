在面向对象编程中，类之间的关系是设计系统结构的核心。根据公开资料，Java 中类的六种主要关系为：继承、实现、依赖、关联、聚合、组合。这些关系可按语义强度排序为：

> 依赖 < 关联 < 聚合 < 组合  
> 继承 & 实现 属于纵向（泛化）关系，其余为横向（引用）关系。

---

一、六种关系的联系与区别

- 继承（Inheritance）  
  - 表示“is-a”关系，子类继承父类的属性和方法。  
  - Java 通过 `extends` 关键字实现。  
  - 示例：`Dog extends Animal`

- 实现（Realization）  
  - 表示类与接口之间的“实现契约”关系。  
  - Java 通过 `implements` 关键字实现。  
  - 示例：`UserService implements IUserService`

- 依赖（Dependency）  
  - “uses-a”关系，临时性、弱依赖，B 的变化影响 A。  
  - 通常表现为：方法参数、局部变量或静态方法调用。  
  - 示例：`person.writeCode(computer)`，其中 `computer` 是局部变量。

- 关联（Association）  
  - “has-a”的长期、稳定关系，双方平等，可单向或双向。  
  - 通过成员变量引用对方。  
  - 示例：`Person` 持有 `Shoes` 对象。

- 聚合（Aggregation）  
  - 弱整体-部分关系，部分可脱离整体存在（如汽车与轮胎）。  
  - 部件可被多个整体共享，生命周期独立。  
  - 代码上与关联相同，语义区分。

- 组合（Composition）  
  - 强整体-部分关系，部分不能脱离整体（如人与大脑）。  
  - 整体负责部分的创建与销毁，生命周期一致。  
  - 通常在构造函数中通过 `new` 实例化部分。

> ✅ 关键区别：  
> - 继承/实现 是纵向扩展；  
> - 依赖/关联/聚合/组合 是横向引用，强度依次增强；  
> - 聚合 vs 组合：看部分是否能独立存在、是否共享、生命周期是否绑定。

---

二、具体代码示例（Java）

```java
// 1. 继承
class Animal {}
class Dog extends Animal {} // 子类继承父类

// 2. 实现
interface Flyable {
    void fly();
}
class Bird implements Flyable { // 类实现接口
    public void fly() { System.out.println("Flying"); }
}

// 3. 依赖
class Person {
    void writeCode(Computer c) { // Computer 作为方法参数
        System.out.println("Writing code with " + c);
    }
}
class Computer {}

// 4. 关联
class Person {
    private Shoes shoes; // 成员变量，长期引用
    public void setShoes(Shoes shoes) { this.shoes = shoes; }
}
class Shoes {}

// 5. 聚合
class Engine {}
class Car {
    private Engine engine; // 聚合：Engine 可独立存在
    public Car(Engine engine) { this.engine = engine; } // 通过参数注入
}

// 6. 组合
class Heart {}
class Person {
    private Heart heart; // 组合：Heart 不能脱离 Person
    public Person() {
        this.heart = new Heart(); // 在构造函数中创建
    }
}
```
