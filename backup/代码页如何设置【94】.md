代码页（Code Page）是字符编码的一种实现方式，通过数字编号定义字节值与字符之间的映射关系。其设置与规定主要涉及操作系统、应用程序和系统配置等多个层面。

---

代码页的设置方式

- Windows 系统层面  
  - 通过 区域设置（Region & Language）自动关联默认代码页。例如：
    - 简体中文系统默认使用 代码页 936（GBK）
    - 英文系统默认使用 代码页 1252（ANSI Latin-1）
  - 可在“控制面板” → “区域” → “管理”选项卡中更改“非 Unicode 程序的语言”，从而修改系统默认 ANSI 代码页 。

- 命令行（CMD）临时设置  
  - 使用 `chcp` 命令查看或切换当前控制台代码页：
    ```cmd
    chcp        查看当前代码页
    chcp 65001  切换到 UTF-8（Windows 中的 UTF-8 对应代码页 65001）
    chcp 936    切换到简体中文 GBK
    ```
  - 也可通过 `mode con cp select=yyy` 设置 OEM 代码页（如 437、850、936 等）。

- 程序内动态设置  
  - C/C++：调用 `SetConsoleOutputCP(936)` 设置控制台输出代码页；使用 `setlocale(LC_ALL, "")` 应用系统区域设置 。
  - Python：在脚本开头声明编码（如 `-*- coding: utf-8 -*-`），或使用 `open(file, encoding='gbk')` 指定读取编码 。
  - ASP/ActiveX：通过 `<%@ CODEPAGE=936 %>` 或 `Session.CodePage = 936` 设置页面代码页 。

- 应用清单声明（Windows 10 1903+）  
  - 在 AppX 或 Win32 应用的清单文件中添加 `<activeCodePage>UTF-8</activeCodePage>`，使进程默认使用 UTF-8 。

---

代码页的规定与标准

- 历史由 IBM 和 Microsoft 定义  
  - 最早由 IBM 在 BIOS 层面引入（OEM 代码页，如 437、850、866）。
  - Microsoft 后续扩展了 ANSI 代码页（如 1252、936、950），用于图形界面应用 。

- 编码类型分类  
  - 单字节代码页（1 字节/字符）：最多 256 个字符，适用于西欧语言（如 1252）。
  - 双字节代码页（DBCS）：用于中文、日文、韩文等（如 936、932、949），可表示超过 65,000 个字符 。
  - Unicode 代码页：如 65001（UTF-8）、1200（UTF-16），用于统一多语言支持 。

- 兼容性与局限  
  - 所有代码页共享 ASCII 0–127（0x00–0x7F）。
  - 跨代码页交换数据可能导致 乱码或字符丢失，因扩展字符（如 é、汉）在不同代码页中映射不同 。
  - 因此，现代系统推荐使用 UTF-8（代码页 65001）作为统一编码 。

---

当前趋势：向 UTF-8 迁移

- Windows 10 1903 及更高版本支持将系统默认代码页设为 UTF-8（需启用 Beta 功能）。
- 新开发应优先使用 Unicode（UTF-8/UTF-16），避免代码页转换问题 。
- 数据库（如 SQL Server）推荐使用 `nchar`/`nvarchar` 等 Unicode 类型，而非依赖代码页的 `char`/`varchar` 。

> 💡 提示：若需查看 Windows 支持的所有代码页列表，可参考 [Microsoft 官方文档](https://learn.microsoft.com/en-us/windows/win32/intl/code-page-identifiers)。
