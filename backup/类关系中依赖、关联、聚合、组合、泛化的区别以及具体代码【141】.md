在面向对象编程中，类之间的关系是设计系统结构的核心。根据权威公开资料（主要参考阿里云、腾讯云开发者社区及UML标准），依赖、关联、聚合、组合、泛化是五种关键的类关系，它们在语义强度、生命周期和代码体现上各有不同。

---

核心区别概览

- 耦合强度排序（由弱到强）：  
  依赖 < 关联 < 聚合 < 组合 < 泛化

- 语义本质：
  - 依赖：使用关系（“uses-a”）
  - 关联：拥有关系（“has-a”，平等层次）
  - 聚合：整体-部分关系（“has-a”，部分可独立存在）
  - 组合：强整体-部分关系（“contains-a”，部分不可独立存在）
  - 泛化：继承关系（“is-a”）

---

详细说明与代码示例

1. 依赖（Dependency）
- 语义：一个类临时使用另一个类，通常作为方法参数、局部变量或静态方法调用。
- 生命周期：短暂，随方法调用而生，随方法结束而灭。
- 代码体现（Java）：
  ```java
  class Book {
      public void borrow(Library lib) { // Library 作为方法参数
          System.out.println("借书 from " + lib.name);
      }
  }

  class Library {
      String name;
      public Library(String name) { this.name = name; }
  }
  ```
- UML表示：虚线 + 箭头（`Book` → `Library`）。

---

2. 关联（Association）
- 语义：两个类长期持有彼此引用，属于“has-a”关系，且处于同一层次。
- 生命周期：与对象实例一致，对象存在则关系存在。
- 代码体现（Java）：
  ```java
  class Teacher {
      private Student student; // 成员变量
      public Teacher(Student s) { this.student = s; }
  }

  class Student {
      private String name;
      public Student(String name) { this.name = name; }
  }
  ```
- UML表示：实线 + 箭头（可单向或双向）。

---

3. 聚合（Aggregation）
- 语义：整体-部分关系，部分可以脱离整体独立存在。
- 生命周期：整体与部分独立管理，删除整体不影响部分。
- 代码体现（Java）：
  ```java
  class Car {
      private Engine engine; // 部分可独立存在
      public Car(Engine e) { this.engine = e; } // 通过构造注入
  }

  class Engine {
      public void start() { System.out.println("Engine started"); }
  }

  // 使用示例
  Engine myEngine = new Engine();
  Car car1 = new Car(myEngine);
  Car car2 = new Car(myEngine); // 同一引擎可被多个车共享
  ```
- UML表示：空心菱形 + 实线（菱形指向整体 `Car`）。

---

4. 组合（Composition）
- 语义：强整体-部分关系，部分不能脱离整体存在，生命周期完全由整体控制。
- 生命周期：整体创建部分，整体销毁时部分也被销毁。
- 代码体现（Java）：
  ```java
  class House {
      private Room room; // 房间依赖于房子
      public House() {
          this.room = new Room(); // 在构造函数中创建
      }
  }

  class Room {
      public Room() { System.out.println("Room created"); }
  }

  // 使用示例
  House house = new House(); // House 创建 Room
  // 当 house 被 GC，room 也随之销毁
  ```
- UML表示：实心菱形 + 实线（菱形指向整体 `House`）。

---

5. 泛化（Generalization）
- 语义：继承关系，子类“是一种”父类（“is-a”）。
- 代码体现（Java）：
  ```java
  class Animal {
      void eat() { System.out.println("Eating..."); }
  }

  class Dog extends Animal { // 泛化关系
      void bark() { System.out.println("Barking..."); }
  }
  ```
- UML表示：实线 + 空心三角箭头（指向父类）。

---

关键总结

| 关系 | 语义 | 生命周期 | 代码形式 | UML 图形 |
|------|------|----------|----------|----------|
| 依赖 | 使用 | 方法级 | 局部变量/参数/静态调用 | 虚线 + 箭头 |
| 关联 | 拥有（平等） | 对象级 | 成员变量 | 实线 + 箭头 |
| 聚合 | 整体-部分（可分离） | 独立 | 成员变量（通过参数注入） | 空心菱形 + 实线 |
| 组合 | 整体-部分（强绑定） | 同生共死 | 成员变量（通过 `new` 创建） | 实心菱形 + 实线 |
| 泛化 | 继承（is-a） | 类级 | `extends` | 实线 + 空心三角 |

> 💡 提示：聚合与组合在语法上完全相同（都是成员变量），区别仅在于语义和生命周期责任，需结合业务上下文判断 。
