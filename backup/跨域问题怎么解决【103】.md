跨域问题主要通过CORS、代理服务器、JSONP等方式解决，其中CORS 是目前最主流的方案。
主流解决方案

1. CORS（跨域资源共享）：通过后端设置响应头（如`Access-Control-Allow-Origin`）允许跨域请求，支持所有 HTTP 方法，需后端配合配置 。
2. 代理服务器：
   - Nginx 反向代理：配置服务器转发请求，隐藏真实后端地址，前端无需修改 。
   - Node.js 中间件：开发环境下使用 webpack-dev-server 等工具代理请求 。
   - 网关层：Spring Cloud Gateway 等网关统一配置跨域规则 。
其他特定场景方案

1. JSONP：利用`<script>`标签无跨域限制特性，仅支持 GET 请求，存在安全隐患，不推荐新项目使用 。
2. postMessage：适用于 iframe 或窗口间通信，需双方配合监听消息 。
3. WebSocket：建立持久连接实现双向通信，不受同源策略限制 。
选择建议

- 生产环境：优先使用CORS或Nginx 代理，安全性高且兼容性好。
- 开发环境：可使用本地代理快速调试。
- 老旧系统：若仅涉及 GET 请求且无法修改后端，可考虑 JSONP 过渡 。
