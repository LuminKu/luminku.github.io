🔧 命令本质与来源

🏛️ 历史演进：ifconfig → ip

特性 ifconfig (已废弃) ip (现代标准)
📦 所属包 net-tools (传统) iproute2 (现代)
🆕 维护状态 ❌ 不再维护 (2001年起) ✅ actively maintained
🔧 功能丰富度 基础 极其丰富 (路由、隧道、策略等)
⚡ 性能 一般 更快，使用 Netlink 套接字

💡 关键认知：ip 命令是 Linux 内核网络子系统的标准接口，不是第三方扩展！它通过 Netlink 协议 直接与内核通信，绕过 /proc 文件系统，效率更高。

🧩 软件包归属

# 查看 ip 命令属于哪个包
dpkg -S $(which ip)
# 输出：iproute2: /usr/sbin/ip

# 查看包信息
apt show iproute2

iproute2 是 Linux 网络工具的事实标准，包含：

  • ip - 网络设备/地址/路由管理
  • ss - 套接字统计 (替代 netstat)
  • tc - 流量控制 (Traffic Control)
  • bridge - 网桥管理
  • rtmon - 路由监控

🔬 逐字段深度拆解

1️⃣ 接口标识行解析

3: wlp2s0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc noqueue state UP group default qlen 1000
字段 值 含义详解
3 接口索引 内核分配给接口的唯一数字标识，用于快速查找
wlp2s0 接口名 Predictable Network Interface Names 命名规则：
• wl = wireless (无线)
• p2 = PCI 总线 2
• s0 = 插槽 0
<...> 标志位 接口的硬件/软件状态标志 (见下表)
mtu 1500 最大传输单元 以太网标准 MTU，超过此值需分片
qdisc noqueue 队列规则 无队列调度 (常用于虚拟接口)，物理网卡通常是 fq_codel 或 pfifo_fast
state UP 接口状态 administratively UP (已启用)
group default 接口组 逻辑分组，便于批量管理
qlen 1000 发送队列长度 待发送数据包的最大排队数

🚩 标志位详解 (FLAGS)

<BROADCAST, MULTICAST, UP, LOWER_UP>
     │         │        │      │
     │         │        │      └─ 物理层已连接 (有载波信号)
     │         │        └─ 接口已启用 (ifconfig up / ip link set up)
     │         └─ 支持多播 (Multicast)
     └─ 支持广播 (Broadcast，以太网标准)

对比你的接口：

接口 标志 状态解读
lo <LOOPBACK,UP,LOWER_UP> 🟢 本地回环，始终活跃
eno1 <NO-CARRIER,BROADCAST,MULTICAST,UP> 🔴 已启用但 无网线连接 (NO-CARRIER)
wlp2s0 <BROADCAST,MULTICAST,UP,LOWER_UP> 🟢 无线网卡已连接 WiFi
docker0 <NO-CARRIER,BROADCAST,MULTICAST,UP> 🟡 网桥已创建，但无容器连接
br-3021998f6676 <BROADCAST,MULTICAST,UP,LOWER_UP> 🟢 Docker Compose 网桥，有容器正在使用
veth* <BROADCAST,MULTICAST,UP,LOWER_UP> 🟢 虚拟网卡对，连接容器与网桥

2️⃣ 链路层信息 (L2)

link/ether ac:fd:ce:47:f5:45 brd ff:ff:ff:ff:ff:ff
字段 示例值 含义
link/ether ac:fd:ce:47:f5:45 MAC 地址 (Media Access Control)
• 前 3 字节 ac:fd:ce = 厂商标识 (Intel)
• 后 3 字节 = 设备唯一标识
brd ff:ff:ff:ff:ff:ff 广播地址，以太网广播目标

特殊案例 - 你的 veth 接口：

6: vethf656c41@if2: ... link-netnsid 0
特殊字段 含义
@if2 这是 veth pair 的一端，另一端连接到索引为 2 的接口 (在容器的网络命名空间中)
link-netnsid 0 标识对端所在的 网络命名空间 ID

3️⃣ 网络层信息 (L3) - IPv4

inet 192.168.0.105/24 brd 192.168.0.255 scope global dynamic noprefixroute wlp2s0
   valid_lft 5192sec preferred_lft 5192sec
字段 值 深度解析
inet 192.168.0.105/24 IPv4 地址 + CIDR 前缀长度 (24 = 255.255.255.0)
brd 192.168.0.255 子网广播地址 (192.168.0.0/24 的广播)
scope global 地址作用域：
• host - 仅本机 (如 127.0.0.1)
• link - 仅链路本地
• global - 全局可路由
dynamic - 通过 DHCP 动态获取 (非静态配置)
noprefixroute - 不自动添加对应子网的路由 (systemd-networkd 特性)
wlp2s0 接口名 地址所属的接口 (当一行显示多个地址时有用)
valid_lft 5192sec 有效生命周期 (DHCP 租约剩余时间 ≈ 1.4 小时)
preferred_lft 5192sec 优先生命周期 (在此时间前，该地址是首选源地址)

⏰ 生命周期解读：你的 WiFi 通过 DHCP 获取 IP，租约即将到期 (5192秒 ≈ 1小时26分)，到期前会自动续租。

对比静态配置 (如 docker0)：

inet 172.17.0.1/16 brd 172.17.255.255 scope global docker0
   valid_lft forever preferred_lft forever
  • forever = 永久有效，静态配置
  • 无 dynamic 标志

4️⃣ 网络层信息 (L3) - IPv6

inet6 fe80::ad37:46f0:87ac:74da/64 scope link noprefixroute 
   valid_lft forever preferred_lft forever
字段 值 含义
fe80::... 链路本地地址 以 fe80::/10 开头，仅在本地链路通信，不可路由
/64 前缀长度 IPv6 标准子网大小
scope link 链路作用域 仅在本网段有效

🎯 实战案例分析

案例 1：网络故障排查 🔍

场景：无法上网，但 WiFi 显示已连接

# 检查步骤：
ip addr show wlp2s0

# 关键观察点：
# 1. 是否有 <LOWER_UP> 标志？没有 = 物理层问题
# 2. 是否有 inet 地址？没有 = DHCP 失败
# 3. valid_lft 是否为 0？是 = IP 已过期

你的 eno1 案例：

2: eno1: <NO-CARRIER,BROADCAST,MULTICAST,UP> ... state DOWN
  • UP + NO-CARRIER = 网卡已启用，但 网线未插或交换机断电
  • state DOWN = 内核认为接口未就绪

案例 3：预测性接口命名规则 🔮

你的接口名 wlp2s0 和 eno1 遵循 systemd 的可预测命名规则：

旧名称 新名称 规则 含义
eth0 eno1 en+o+1 Ethernet on-board NIC #1
wlan0 wlp2s0 wl+p2+s0 Wireless PCI bus 2 Slot 0

优点：插拔网卡后名称不会乱变！

# 查看命名策略
cat /etc/systemd/network/99-default.link
# 或
udevadm info /sys/class/net/wlp2s0

🚀 高级用法与技巧

🎨 格式化输出 (JSON)

# 现代 iproute2 支持 JSON 输出 (便于脚本处理)
ip -json addr show wlp2s0 | jq '.[0].addr_info'

🔍 过滤特定接口

# 只显示活跃的接口
ip addr show up

# 只显示特定接口
ip addr show dev wlp2s0

# 按 IP 地址查找接口
ip addr show to 192.168.0.0/24

⚡ 临时修改 (立即生效，重启丢失)

# 添加 IP 地址
sudo ip addr add 192.168.1.100/24 dev eno1

# 删除 IP 地址
sudo ip addr del 192.168.1.100/24 dev eno1

# 启用/禁用接口
sudo ip link set eno1 up
sudo ip link set eno1 down

📊 监控接口变化

# 实时监控地址变化 (类似 tail -f)
ip monitor addr

# 监控链路状态
ip monitor link
