要让显卡发挥作用，关键在于调整一个核心参数——`OLLAMA_NUM_GPU_LAYERS`，它能精细地控制模型在GPU和CPU之间的任务分配。为了平衡你的性能需求和显卡6GB显存的限制，我为你准备了四套方案，你可以从“快速验证”开始，逐步测试，找到最适合你电脑的配置。

### ⚙️ 配置修改小提示

在开始前，有一点配置设定需要留意：在Windows系统中设置的环境变量，需要**重启你电脑上的Ollama应用**才能生效。每次修改后都需要重复这个步骤。

### 方案一：CPU模式（🛡️备用/去险方案）
这个方案主要是为了让你了解如何强制在CPU上运行，虽然速度最慢，但在多任务处理或遇到兼容性问题时，它是一个可靠的备用选择。

*   **配置代码**：
    ```shell
    # 设置环境变量
    set CUDA_VISIBLE_DEVICES=-1
    set OLLAMA_CONTEXT_LENGTH=2048

    # 重启Ollama后，运行模型
    ollama run deepseek-r1:8b
    ```
*   **核心参数解读**：
    *   `CUDA_VISIBLE_DEVICES=-1`：这是核心指令，会告诉Ollama不要去检测和使用任何NVIDIA GPU。
    *   `OLLAMA_CONTEXT_LENGTH=2048`：将上下文窗口设置为2048，可以减少内存占用，对CPU模式比较友好。

---

### 方案二：快速验证模式（🚀快速验证方案）
这个方案适合快速验证配置是否生效，速度比纯CPU模式快不少，能让你立刻看到效果。

*   **配置代码**：
    ```shell
    # 设置环境变量
    set OLLAMA_NUM_GPU_LAYERS=20
    set OLLAMA_CONTEXT_LENGTH=2048
    set OLLAMA_FLASH_ATTENTION=1

    # 重启Ollama后，运行模型
    ollama run deepseek-r1:8b
    ```
*   **核心参数解读**：
    *   `OLLAMA_NUM_GPU_LAYERS=20`：将模型的前20层放入GPU（约占总计算量的60%），是混合分配的核心参数。
    *   `OLLAMA_FLASH_ATTENTION=1`：这个指令能为你的RTX 3060启用Flash Attention，这是一种能显著提升推理速度并降低显存占用的技术。有时一条命令就能解决很多问题。

### 方案三：性能优先-激进模式（🚀激进/推荐方案）
这个方案进一步提高了GPU的计算比重，为你的6000系N卡（RTX 3060）做了Flash Attention和最大并行请求数的优化，以追求极致的生成速度。

*   **配置代码**：
    ```shell
    # 设置环境变量
    set OLLAMA_NUM_GPU_LAYERS=33
    set OLLAMA_CONTEXT_LENGTH=1024
    set OLLAMA_FLASH_ATTENTION=1
    set OLLAMA_NUM_PARALLEL=4

    # 重启Ollama后，运行模型
    ollama run deepseek-r1:8b
    ```
*   **核心参数解读**：
    *   `OLLAMA_NUM_GPU_LAYERS=33`：将绝大部分计算（33层）交给RTX 3060。
    *   `OLLAMA_NUM_PARALLEL=4`：启用4个并行请求，可以有效压榨GPU性能，提高吞吐量。

### 方案四：稳定性优先-保守模式（🛡️保守/稳定方案）
这个方案在追求速度的同时，通过增加CPU的介入来保证系统的整体稳定性，避免因显存不足导致程序崩溃。

*   **配置代码**：
    ```shell
    # 设置环境变量
    set OLLAMA_NUM_GPU_LAYERS=10
    set OLLAMA_CONTEXT_LENGTH=4096
    set OLLAMA_FLASH_ATTENTION=1

    # 重启Ollama后，运行模型
    ollama run deepseek-r1:8b
    ```
*   **核心参数解读**：
    *   `OLLAMA_CONTEXT_LENGTH=4096`：用回4096的标准上下文窗口。
    *   `OLLAMA_NUM_GPU_LAYERS=10`：只将轻量级任务给GPU（10层），这在运行长文本时能防止因显存溢出（OOM）导致的应用崩溃。

### 💡 追问：关于"共享GPU内存"和"双GPU"的补充解释

#### ❓ 为什么有"共享内存"却用不上？
你看到的`总GPU内存 = 专用显存 + 共享GPU内存`，但其中**起决定性作用的是专用显存（6GB）**。这个"共享内存"机制是：**当专用显存用尽后，系统才会启用**，但它依然会占用大量系统内存并引发CPU-GPU数据交换。性能虽不如纯靠专用显存，但遇到超大模型（14B）时可以作为备选方案。

#### ❓双GPU（RTX 3060 + AMD集成显卡）能用上吗？
你也可以强制让Ollama同时使用这两块显卡：
1.  确认Ollama服务已停止。
2.  在系统环境变量中设置：
    *   `CUDA_VISIBLE_DEVICES`：`0`（只留RTX 3060的编号）
    *   `OLLAMA_SCHED_SPREAD`：`1`（强制平均分配到所有可见GPU）
3.  重启Ollama服务。

> 需要注意，两张显卡性能差异巨大，混合使用可能导致性能瓶颈反而不如单张RTX 3060流畅。
