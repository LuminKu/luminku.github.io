以下是 依赖、关联、聚合、组合、泛化 五种类间关系的区别总结，包含 语义说明、UML 表示法、代码体现、生命周期特点 以及 典型示例，并附上对应 Java 代码。

---

核心区别对比表

| 关系类型 | 语义描述 | UML 表示法 | 代码体现 | 生命周期特点 | 耦合强度 | 示例 |
|----------|----------|------------|----------|--------------|----------|------|
| 依赖 (Dependency) | “使用”关系，临时性、偶然性 | 虚线 + 箭头（→） | 局部变量、方法参数、静态方法调用 | 方法调用期间存在，方法结束即销毁 | 最弱 | `Student` 使用 `Computer` 写代码 |
| 关联 (Association) | “拥有”关系，长期稳定 | 实线 + 箭头（→） | 成员变量（单向/双向） | 对象创建时建立，销毁时结束 | 较弱 | `Teacher` 持有 `Student` 引用 |
| 聚合 (Aggregation) | “整体-部分”，部分可独立存在 | 空心菱形 + 实线（◇—） | 成员变量，通过构造函数注入 | 整体与部分独立生命周期 | 中等 | `Car` 拥有 `Engine`，但引擎可更换 |
| 组合 (Composition) | “整体-部分”，部分不能独立存在 | 实心菱形 + 实线（◆—） | 成员变量，在构造函数中 `new` | 部分随整体生灭 | 强 | `Person` 拥有 `Heart`，心脏不能脱离人体 |
| 泛化 (Generalization) | “is-a” 继承关系 | 实线 + 空心三角（△—） | `extends`（类）或 `implements`（接口） | 子类继承父类结构与行为 | 最强 | `Cat` 继承 `Animal` |

> 注：实现（Realization）关系（类对接口）通常与泛化归为一类，用虚线 + 空心三角表示（如 `class Dog implements Animal`）。

---

具体 Java 代码示例

1. 依赖 (Dependency)
```java
class Library {}
class Book {
    void borrow() {
        Library lib = new Library(); // 局部变量 → 依赖
        lib.checkOut();
    }
}
```

2. 关联 (Association)
```java
class Student {}
class Teacher {
    private Student student; // 成员变量 → 关联
    public Teacher(Student s) { this.student = s; }
}
```

3. 聚合 (Aggregation)
```java
class Engine {}
class Car {
    private Engine engine; // 成员变量 → 聚合
    public Car(Engine e) { this.engine = e; } // 通过参数注入
}
```

4. 组合 (Composition)
```java
class Heart {}
class Person {
    private Heart heart; // 成员变量 → 组合
    public Person() { this.heart = new Heart(); } // 在构造函数中 new
}
```

5. 泛化 (Generalization)
```java
class Animal {}
class Cat extends Animal { } // extends → 泛化
```

> 若涉及接口实现（如 `implements`），则属于 实现关系，也归入广义泛化范畴：
> ```java
> interface Fly { void fly(); }
> class Bird implements Fly { public void fly() { ... } } // 实现关系
> ```

---

补充说明

- 聚合 vs 组合：两者在代码形式上几乎相同（均为成员变量），关键区别在于 语义与生命周期。聚合强调“可替换”，组合强调“不可分割”。
- 关联 vs 依赖：关联是“拥有”（成员变量），依赖是“使用”（临时引用）。
- 耦合强度排序（由弱到强）：  
  依赖 < 关联 < 聚合 < 组合 < 泛化/实现 。
