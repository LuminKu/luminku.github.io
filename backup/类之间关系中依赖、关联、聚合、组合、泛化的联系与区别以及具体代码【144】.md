在面向对象设计中，类之间的关系是建模系统结构的核心。根据 UML（统一建模语言）和 Java 语言实践，依赖、关联、聚合、组合、泛化 是五种关键关系，它们在语义强度、生命周期和代码表现上各有不同。

---

一、五种关系的联系与区别

- 耦合强度排序（由弱到强）：  
  依赖 < 关联 < 聚合 < 组合 < 泛化

- 本质分类：
  - 使用关系（Use-a）：依赖
  - 拥有关系（Has-a）：关联、聚合、组合
  - 继承关系（Is-a）：泛化、实现（虽未在问题中列出，但常并列讨论）

- 关键区别点：
  - 依赖：临时使用，方法内局部变量或参数。
  - 关联：长期持有，成员变量，双方通常处于同一层次。
  - 聚合：整体-部分，部分可独立存在（如班级与学生）。
  - 组合：整体-部分，部分不能独立存在，生命周期由整体控制（如人与心脏）。
  - 泛化：继承，子类扩展父类，属于“is-a”关系。

> ⚠️ 注意：聚合与组合在代码实现上完全相同（均为成员变量），只能通过语义和业务逻辑区分 。

---

二、具体代码示例（Java）

1. 依赖（Dependency）
- 表现：方法内局部变量、参数、静态方法调用。
- 语义：“A 使用 B 来完成某操作”，非持久持有。

```java
class Library {
    public void borrowBook() { System.out.println("借书"); }
}

class Student {
    public void study() {
        Library lib = new Library(); // 局部变量 → 依赖
        lib.borrowBook();
    }
}
```

2. 关联（Association）
- 表现：成员变量，单向或双向。
- 语义：“A 拥有 B 的引用”，长期稳定关系。

```java
class Teacher {
    private String name;
}

class Student {
    private Teacher teacher; // 成员变量 → 关联
    public Student(Teacher t) { this.teacher = t; }
}
```

3. 聚合（Aggregation）
- 表现：成员变量，通过构造器注入。
- 语义：“整体包含部分，但部分可独立存在”。

```java
class Engine {
    private String type;
    public Engine(String type) { this.type = type; }
}

class Car {
    private Engine engine;
    public Car(Engine e) { this.engine = e; } // 聚合：Engine 可被多个 Car 共享或独立存在
}
```

4. 组合（Composition）
- 表现：成员变量，在构造器中 `new` 出来。
- 语义：“部分是整体不可分割的一部分，同生共死”。

```java
class Heart {
    public Heart() { System.out.println("心脏创建"); }
}

class Person {
    private Heart heart;
    public Person() {
        this.heart = new Heart(); // 组合：Heart 生命周期由 Person 控制
    }
}
```

5. 泛化（Generalization）
- 表现：`extends`（类继承）或 `implements`（接口实现）。
- 语义：“is-a” 关系，子类继承父类行为。

```java
class Animal {
    public void eat() { System.out.println("吃东西"); }
}

class Dog extends Animal { // 泛化：Dog 是 Animal 的一种
    public void bark() { System.out.println("汪汪"); }
}
```

---

三、补充说明：实现关系（Realization）

虽然问题未明确要求，但常与泛化并列：

```java
interface Flyable {
    void fly();
}

class Bird implements Flyable { // 实现关系：类实现接口
    public void fly() { System.out.println("鸟在飞"); }
}
```

四、总结对比表

| 关系     | 代码形式               | 生命周期       | 语义         | UML 图形表示         |
|----------|------------------------|----------------|--------------|----------------------|
| 依赖     | 局部变量 / 参数        | 方法调用期间   | 使用         | 虚线 + 箭头          |
| 关联     | 成员变量               | 对象存在期间   | 拥有         | 实线 + 箭头          |
| 聚合     | 成员变量（构造注入）   | 可独立         | 弱整体-部分  | 空心菱形 + 实线      |
| 组合     | 成员变量（构造中 new） | 同生共死       | 强整体-部分  | 实心菱形 + 实线      |
| 泛化     | `extends` / `implements` | 继承结构       | is-a         | 实线/虚线 + 空心三角 |
