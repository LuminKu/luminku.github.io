在面向对象编程中，类关系描述了不同类之间的交互与结构联系。根据主流资料（如腾讯云、CSDN、博客园等权威平台），类关系通常分为六种：继承（泛化）、实现、依赖、关联、聚合、组合。这些关系按耦合强度从弱到强排序为：

> 依赖 < 关联 < 聚合 < 组合 < 继承/实现

---

六种类关系详解

1. 继承（Generalization）
- 含义：“is-a”关系，子类继承父类的属性和方法。
- 代码体现：Java/C++ 中使用 `extends`。
- UML 表示：实线 + 空心三角箭头（指向父类）。
- 示例：
  ```java
  class Animal {}
  class Dog extends Animal {} // Dog 是一种 Animal
  ```

2. 实现（Realization）
- 含义：类实现接口定义的行为契约。
- 代码体现：Java 中使用 `implements`。
- UML 表示：虚线 + 空心三角箭头（指向接口）。
- 示例：
  ```java
  interface Flyable { void fly(); }
  class Bird implements Flyable {
      public void fly() { System.out.println("Bird is flying"); }
  }
  ```

3. 依赖（Dependency）
- 含义：“use-a”关系，临时使用另一个类，耦合最弱。
- 代码体现：方法参数、局部变量、静态方法调用。
- UML 表示：虚线 + 箭头。
- 示例：
  ```java
  class Person {
      void writeCode(Computer c) { // Computer 作为参数
          System.out.println("Writing code...");
      }
  }
  class Computer {}
  ```

4. 关联（Association）
- 含义：“has-a”关系，长期持有对方引用，结构化连接。
- 代码体现：成员变量持有其他类对象。
- UML 表示：实线 + 箭头（可单向或双向）。
- 示例：
  ```java
  class Student {
      private Teacher teacher; // 成员变量
      public void setTeacher(Teacher t) { this.teacher = t; }
  }
  class Teacher {}
  ```

5. 聚合（Aggregation）
- 含义：弱“has-a”，整体与部分可独立存在，部分可共享。
- 代码体现：成员变量（通常通过构造器或 Setter 注入）。
- UML 表示：实线 + 空心菱形（指向整体）。
- 示例：
  ```java
  class Department {
      private List<Employee> employees;
      public Department(List<Employee> emps) { this.employees = emps; }
  }
  class Employee {} // 可独立于 Department 存在
  ```

6. 组合（Composition）
- 含义：强“has-a”，整体拥有部分生命周期，部分不能独立存在。
- 代码体现：成员变量在整体内部通过 `new` 创建。
- UML 表示：实线 + 实心菱形（指向整体）。
- 示例：
  ```java
  class Human {
      private Heart heart = new Heart(); // 心脏随人体创建和销毁
  }
  class Heart {}
  ```

---

联系与区别总结

| 关系类型 | 生命周期依赖 | 典型场景 | 代码形式 | 耦合强度 |
|----------|--------------|----------|----------|----------|
| 依赖 | 无 | 临时使用（如工具类） | 方法参数、局部变量 | 最弱 |
| 关联 | 无 | 长期引用（如用户-地址） | 成员变量（引用） | 弱 |
| 聚合 | 无 | 整体-部分可分离（如公司-员工） | 成员变量（外部注入） | 中 |
| 组合 | 有（整体控制部分） | 整体-部分共生死（如订单-订单项） | 成员变量（内部 `new`） | 强 |
| 继承 | 有（编译绑定） | “is-a”复用（如 Dog-Animal） | `extends` | 很强 |
| 实现 | 有（契约绑定） | 实现接口规范（如 Service-Impl） | `implements` | 强 |

> 💡 关键区分点：
> - 聚合 vs 组合：聚合中部分可独立存在（如轮胎可换车）；组合中部分不能脱离整体（如大脑不能脱离人体）。
> - 关联 vs 依赖：关联是“拥有”（成员变量），依赖是“使用”（方法内临时使用）。

---

实战建议
- 优先使用组合/聚合而非继承，降低耦合。
- 接口设计应面向实现，提高扩展性。
- 避免双向关联导致循环引用，必要时用 `weak reference`（如 Java 中的 `WeakReference`）。
