类关系的联系与区别以及具体代码【147】

在面向对象编程中，类之间的关系是构建清晰、可维护系统的核心。根据权威公开资料（如CSDN、博客园等平台的多篇技术文章），类关系主要分为六种：依赖、关联、聚合、组合、继承（泛化）、实现。它们按耦合强度从弱到强排序为：

依赖 < 关联 < 聚合 < 组合 < 继承/实现

一、六种类关系的联系与区别

  • 继承（泛化） 与 实现 属于纵向关系（“is-a”），体现类或接口的层级。
  • 依赖、关联、聚合、组合 属于横向关系（“has-a” 或 “uses-a”），体现类之间的协作或包含。
关系类型 UML 表示 代码体现 生命周期 耦合强度 典型语义
依赖 虚线 + 箭头 方法参数、局部变量、静态方法调用 无（临时） 最弱 “uses-a”（如人用船过河）
关联 实线 + 箭头 成员变量（引用） 无（长期） 弱 “has-a”（如客户有地址）
聚合 空心菱形 + 实线 成员变量（常通过 Setter 或构造注入） 可独立存在 中等 “整体拥有部分，但部分可独立”（如公司有员工）
组合 实心菱形 + 实线 成员变量（在整体内部创建） 整体控制部分，共生死 强 “contains-a”（如人有大脑）
继承 实线 + 空心三角 extends 关键字 有（编译绑定） 很强 “is-a”（如狗是动物）
实现 虚线 + 空心三角 implements 关键字 有（契约绑定） 强 “实现接口”（如服务实现接口）

✅ 关键区分点：

  • 聚合 vs 组合：看部分是否能脱离整体独立存在（如轮胎 vs 引擎）。
  • 关联 vs 依赖：关联是成员变量（长期），依赖是方法参数或局部变量（临时）。
  • 组合/聚合 vs 关联：组合和聚合是关联的特例，强调“整体-部分”关系。

二、具体代码示例（Java）

// 1. 依赖关系：临时使用
class Printer {}
class ReportGenerator {
    void generateReport(Printer printer) { // 方法参数
        printer.print();
    }
}

// 2. 关联关系：长期持有引用
class Address {}
class Customer {
    private Address address; // 成员变量
}

// 3. 聚合关系：整体与部分可分离
class Engine {}
class Car {
    private Engine engine; // 通过构造注入
    public Car(Engine engine) { this.engine = engine; }
}

// 4. 组合关系：整体控制部分生命周期
class Brain {}
class Person {
    private Brain brain;
    public Person() {
        this.brain = new Brain(); // 在内部创建
    }
}

// 5. 继承关系：is-a
class Animal {}
class Dog extends Animal {}

// 6. 实现关系：契约实现
interface UserService {}
class UserServiceImpl implements UserService {}

三、设计建议

  • 优先使用组合 > 继承：避免“继承滥用”，提高灵活性。
  • 依赖倒置：依赖接口而非具体类，降低耦合。
  • 聚合与组合选择：若部分可复用或独立存在 → 聚合；若部分完全依附整体 → 组合。
