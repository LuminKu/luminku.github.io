在面向对象设计中，类之间的关系是UML类图的核心内容。根据权威公开资料（主要参考阿里云、腾讯云开发者社区及UML标准文档），依赖、关联、聚合、组合、泛化是五种关键关系，它们在语义强度、生命周期和代码体现上各有不同。

---

一、关系定义与区别

- 泛化（Generalization）  
  - 表示 “is-a” 继承关系（如：狗 是 一种 动物）。  
  - 子类继承父类的属性和方法，支持多态。  
  - 耦合最强，属于“物理关联”。

- 实现（Realization）  
  - 表示类 实现接口 的契约（如：鸟 实现 飞翔 接口）。  
  - 虽未在问题中列出，但常与泛化并列，属于类与接口的关系。

- 组合（Composition）  
  - 表示 “contains-a” 强整体-部分关系，部分不能脱离整体存在（如：人 与 心脏）。  
  - 生命周期一致：整体销毁，部分也随之销毁。

- 聚合（Aggregation）  
  - 表示 “has-a” 弱整体-部分关系，部分可独立于整体存在（如：班级 与 学生）。  
  - 属于关联的特例，耦合弱于组合。

- 关联（Association）  
  - 表示两个类 长期稳定的“拥有”或“使用”关系，无整体-部分语义（如：老师 与 学生）。  
  - 通过成员变量实现，双向或单向均可。

- 依赖（Dependency）  
  - 表示 临时的“使用”关系，如方法参数、局部变量或静态调用。  
  - 耦合最弱，关系随方法调用结束而终止。

> 耦合强度排序（由强到弱）：  
> 泛化 ≈ 实现 > 组合 > 聚合 > 关联 > 依赖  
> 

---

二、代码示例（Java）

以下代码清晰展示每种关系的实现方式：

1. 泛化（继承）
```java
class Animal { }
class Cat extends Animal { } // extends 表示泛化
```

2. 实现（接口实现）
```java
interface Fly { void fly(); }
class Bird implements Fly { // implements 表示实现
    public void fly() { System.out.println("Flying"); }
}
```

3. 组合（强整体-部分）
```java
class Heart {
    public Heart() { System.out.println("Heart created"); }
}

class Person {
    private Heart heart;
    public Person() {
        heart = new Heart(); // 组合：整体创建部分
    }
}
```

4. 聚合（弱整体-部分）
```java
class Department { }

class Company {
    private Department dept;
    public Company(Department dept) { // 聚合：部分可独立存在
        this.dept = dept;
    }
}
```

5. 关联（长期拥有）
```java
class Teacher {
    private Student student; // 成员变量表示关联
    public Teacher(Student s) { this.student = s; }
}
```

6. 依赖（临时使用）
```java
class Order {
    public void calculateTotal() {
        TaxCalculator calc = new TaxCalculator(); // 局部变量 → 依赖
        double tax = calc.compute();
    }
}
```

三、关键辨析要点

- 关联 vs 聚合 vs 组合  
  三者在代码形式上相同（均为成员变量），需通过语义和生命周期区分：  
  - 关联：平等关系（如人与手机）  
  - 聚合：整体-部分但可分离（如公司与部门）  
  - 组合：整体-部分且共生（如汽车与发动机）

- 依赖的典型场景  
  - 方法参数  
  - 局部变量  
  - 静态方法调用  
  - 返回值类型（部分资料认为）

- UML图示符号  
  - 泛化：实线 + 空心三角箭头（→ 父类）  
  - 实现：虚线 + 空心三角箭头（→ 接口）  
  - 组合：实线 + 实心菱形（→ 整体）  
  - 聚合：实线 + 空心菱形（→ 整体）  
  - 关联：实线（可带箭头）  
  - 依赖：虚线 + 箭头（→ 被依赖类）
