# YML 文件含义

**YML** 实际上是 **YAML**（YAML Ain’t Markup Language）的文件扩展名简写形式，用于存储**结构化数据**的文本文件，常见于各种软件的**配置文件**中，例如 Docker、Kubernetes、GitHub Actions 等。

无论是 *.yml* 还是 *.yaml*，它们的**语法和功能完全相同**，只是扩展名不同，文件内容的解析方式没有差别。

**核心特点**

- **人类可读性高**：语法简洁，缩进表示层级关系，不使用大括号或引号。
- **支持多种数据类型**：对象（键值对）、数组、纯量（字符串、数字、布尔值等）。
- **广泛应用**：配置文件、数据序列化、CI/CD 流水线定义等。

**扩展名差异**

- *.yaml*：符合 YAML 官方标准，现代项目更常用。
- *.yml*：早期文件系统扩展名长度限制（最多 3 个字符）遗留产物，一些旧系统或工具仍在使用。
- **选择建议**： 若无兼容性限制，优先使用 .yaml。 若工具或系统仅支持 .yml，则必须使用 .yml。 团队内部应统一规范，保持一致性。

**简单示例**（无论 *.yml* 还是 *.yaml* 都可使用）：

```yaml
version: "3.8"
services:
 web:
   image: nginx:latest
   ports:
     - "8080:80"
 db:
   image: mysql:5.7
   environment:
     MYSQL_ROOT_PASSWORD: example
```

该文件可作为 Docker Compose 配置，描述了一个包含 **Nginx** 和 **MySQL** 服务的应用。

**总结** YML 指的就是 YAML 文件的简写扩展名，本质上两者没有区别。选择哪种扩展名取决于**项目规范**、**工具兼容性**和**团队习惯**，但在现代开发中 *.yaml* 更为推荐。
